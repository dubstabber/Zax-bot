## What’s already documented and/or implemented

- **FRM16**
  - Spec: `docs/FRM16_FORMAT.md`
  - Tools: `tools/frm16_decode.py`; corpus entrypoint: `tools/batch_extract.py`
  - Status: 366/366 runtime textures are targeted for PNG output under `extracted/textures/`
  - Latest artifact pass: runtime Layer 4 non-bitmask data is decoded as LUT-row byte alpha where applicable, and Layer 5 is decoded as RGB565 RLE overlay data. This fixes false blank/missing pixels in font, shadow, small-color, and editor sequences that were caused by treating Layer 5 as 8-bit palette indices or Layer 4 as 16-bit RLE.

- **FRM + PAL**
  - Evidence: live `Zax.exe` generic frame loader (`FRM_LoadFrame` at 0x4A9340) and palette init path (`Palettes/GamePalette`)
  - Tools: `tools/frm_decode.py`; corpus entrypoint: `tools/batch_extract.py`
  - Status: 361/361 8-bit indexed textures are converted under `extracted/textures_8bit/`; 9/9 PAL files are copied under `extracted/palettes/`. `GamePalette.pal` is BGR on disk and converted to RGB for PNG output.
  - Latest artifact pass: 8-bit source FRM/SEQ layers 2-5 are now exported as alpha buckets (80/60/40/20%) instead of being ignored. IDA confirms the generic five-layer frame loader and CSequence registers `Load/Use 8 Bit Alpha`; this fixes false blank outputs in source animations such as `Fire Left.seq` and `Fire Right.seq`.

- **SEQ16**
  - Spec: `docs/SEQ16_FORMAT.md`
  - Tools: `tools/extract_seq16.py`; corpus entrypoint: `tools/batch_extract.py`
  - Status: 212/212 runtime animation files converted to 7,605 frame PNGs plus 212 metadata JSON files under `extracted/animations/`

- **SEQ**

## Why these weren’t identified earlier (oversight analysis)

- **Format tunnel vision (FRM16/SEQ16 focus)**: Early work prioritized runtime 16-bit formats (FRM16, SEQ16), assuming sources weren’t shipped. This was reinforced by prior notes claiming “Zax ships only FRM16.”
- **Assumptions about importance**: We deprioritized `.seq` (source) and non-rendering formats (ZAX/WAY/STY), which masked the presence of other assets.
- **Incomplete initial survey**: We didn’t perform a comprehensive, case-insensitive extension inventory across both `data/` and `polish/` at the outset. Upper/lowercase variants (e.g., `RDS`/`rds`, `TXT`/`txt`) inflated perceived gaps and hid totals.
- **Documentation drift**: Cleanup removed outdated docs, but the README’s “Project Structure” still referenced deleted files; this mixed signal likely discouraged re-checking inventories for non-16-bit formats like `frm`.
